Real-Data Workflows

This page summarizes the real-data vignette workflow bundled with ggSynteny.

The package supports two pairwise routes:

  • MCScanX: gff + collinearity
  • jcvi: BED + anchors

Both routes produce the same ggsynteny_data object and can be enriched with optional GFF3 annotation for transcript-aware micro-synteny plotting.

Entry points

  • validate_real_synteny_workflow() for MCScanX input
  • validate_jcvi_synteny_workflow() for jcvi input

Checklist

  • Gene IDs must be consistent between pairwise input and annotation.
  • Chromosome names must match between pairwise input and GFF3.
  • All files must use the same genome build.
  • GFF3 should contain transcript, exon, or CDS features if you want feature-aware micro-synteny.

See vignettes/real-data-workflows.Rmd in the source tree for the full runnable template.
